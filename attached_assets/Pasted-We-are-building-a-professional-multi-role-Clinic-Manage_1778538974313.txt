We are building a professional multi-role Clinic Management System with a separate future Patient App. The current system already includes dashboards, live queue management, patients, appointments, staff management, notifications, activity logs, and clinic settings.

The goal now is to restructure and improve the system professionally in phases.

OBJECTIVE:
Separate staff users from patient users and create a professional clinic architecture.

CURRENT PROBLEM:
Patients and staff users are mixed together in the same onboarding flow. This causes patients to appear inside the Staff & Members page, which is incorrect.

GOAL:
Separate the system into:

1. Staff Management App (current system)
2. Patient App (future frontend)

Both applications must share:

* the same backend
* the same database
* the same API
* the same clinic data

---

## STEP 1 — CREATE USER TYPE SYSTEM

Add a `user_type` field to the users table.

Allowed values:

* staff
* patient

Staff users must also have role values:

* clinic_admin
* doctor
* nurse
* receptionist

Patients must NOT have staff roles.

## STEP 3 — ADD GOVERNMENT IDENTIFICATION

All users joining the clinic system must provide either:

* South African ID number
  OR
* Passport number

Add fields:

* government_id_type
* government_id_number

Examples:

government_id_type = "SA_ID"
government_id_number = "09070843561325"

OR

government_id_type = "PASSPORT"
government_id_number = "A12345678"

Validation rules:

* SA ID must contain exactly 13 digits
* Passport numbers may contain letters and numbers

Prevent duplicate registrations using government ID numbers.

## STEP 4 — FIX AUTHENTICATION FLOW

CURRENT ISSUE:
All new accounts automatically become clinic members/staff.

FIX:

### STAFF REGISTRATION FLOW

* create account
* choose role
* enter clinic code
* submit ID/passport
* request approval
* admin approves access

System must create:

user_type = 'staff'

### PATIENT REGISTRATION FLOW

* create patient account
* no clinic code required
* submit ID/passport
* patient profile created

System must create:

user_type = 'patient'

Patients must NEVER appear in clinic staff/member pages.
